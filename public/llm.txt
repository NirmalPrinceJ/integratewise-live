## WHAT WE ARE BUILDING

Glowing Pancake is the first AI-powered Knowledge Workspace that completes the data-back-to-learning
loop with approval-based AI agents. It solves three core problems:

1. GenAI Divide — Most teams cannot use AI effectively because context is scattered
2. Tool Sprawl — Organizations use 10-30+ tools with no unified intelligence layer
3. CS Paradox — Customer Success teams have the most data but the least tooling

## CORE PHILOSOPHY

- "AI That Thinks in Context, Waits for Approvals"
- "Normalize Once, Render Anywhere"
- Dual-context design: Every AI recommendation requires both vendor AND client goal impact
- Approval-based agents: AI proposes, humans approve, system learns

## ARCHITECTURE OVERVIEW

### L0 — Vision Layer
The platform creates a unified intelligence layer across all business tools.
Two feedback loops converge in a Single Source of Truth (SSOT) called "The Spine":
- Loop A: Human-Governed Context (decisions, goals, approvals)
- Loop B: System-Governed Tool Data (automated ingestion, normalization)

### L1 — Platform Architecture
- 15 Context-Aware Workspace Modules
- 14 Universal Cognitive Components
- 8-Stage Processing Pipeline with 4 Accelerators
- 7 Specialized AI Agents

### L2 — System Design
- Truth Storage: Supabase (PostgreSQL + Row Level Security)
- Compute/Edge/Cache: Cloudflare (Workers, Pages, R2, KV)
- Orchestration: n8n (self-hosted at n8n.integratewise.online)
- Database: Neon PostgreSQL (event store)
- Frontend: Vercel (Next.js)
- File Storage: Box, Cloudflare R2
- Documentation: Notion

### L3 — Implementation Layer
AI Agents:
- SuccessPilot: CS playbook execution and health scoring
- ChurnShield: Churn prediction and intervention
- VaultGuard: Data governance and compliance
- DataSentinel: Data quality monitoring
- ArchitectIQ: Integration architecture recommendations
- TemplateForge: Template and document generation
- DealDesk: Deal structuring and pricing optimization
