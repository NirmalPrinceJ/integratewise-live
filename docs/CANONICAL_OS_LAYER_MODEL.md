## ✅ RUNTIME SEQUENCE (NO-MISS)

This is the non-skippable runtime order. Any implementation that violates this ordering is **unconstitutional**.

### Phase 1 — L0: Reality Introduction + Intent
| # | Event | Description |
|---|-------|-------------|
| 1 | `BUCKET_CREATED` | base_buckets: OFF→ADDING (intent declared) |
| 2 | `SOURCE_CONNECTED` | connector credentials or file selected |
| 3 | `INITIAL_DATA_AVAILABLE` | payload staged |
| 4 | `HANDOFF_TO_ADAPTIVE_SPINE` | L0 stops (no truth writes) |

### Phase 2 — L3: Adaptive Spine (Truth + Learning + Readiness)
| # | Event | Description |
|---|-------|-------------|
| 5 | `INGEST_STARTED` | loader receives payload |
| 6 | `EXTRACTED_RECORDS_CREATED` | raw extraction artifacts created (non-SSOT) |
| 7 | `NORMALIZED_TO_SSOT` | truth entities written (SSOT) |
| 8 | `FIELD_OBSERVATIONS_RECORDED` | spine_schema_registry updated |
| 9 | `COMPLETENESS_COMPUTED` | spine_entity_completeness updated |
| 10 | `STABILITY_EVALUATED` | schema stabilized or degraded |
| 11 | `READINESS_DECIDED` | bucket transitions: ADDING→SEEDED→LIVE (or regression) |
| 12 | `READINESS_SIGNALS_EMITTED` | BUCKET_SEEDED / COMPLETENESS_MET / MATURITY_UPGRADED |

### Phase 3 — L1: Workplace Materialization (Pure Work)
| # | Event | Description |
|---|-------|-------------|
| 13 | `MODULE_GUARD_CHECK` | L1 rendering allowed only if bucket eligibility satisfied |
| 14 | `WORKSPACE_RENDERED` | department surfaces appear (Today, Tasks, Entities, Situations) |
| 15 | `HUMAN_EXECUTES_WORK` | actions produce operational outcomes |

### Phase 4 — L2: Cognition (Invoked from L1; Reads only L3 truth)
| # | Event | Description |
|---|-------|-------------|
| 16 | `USER_INVOKES_COGNITION` | Evidence Drawer opened — L2 is on-demand |
| 17 | `EVIDENCE_FABRIC_READ` | L3 truth + lineage + readiness snapshot pulled |
| 18 | `INSIGHT_GENERATED` | evidence-linked only |
| 19 | `ACTION_PROPOSED` | queued for approval |
| 20 | `APPROVAL_GRANTED` | or rejected (audit logged) |
| 21 | `ACTION_EXECUTED` | tool action performed (governed) |
| 22 | `DECISION_RECORDED` | decision memory + replay artifacts written |

### Phase 5 — Loop Closure
| # | Event | Description |
|---|-------|-------------|
| 23 | `NEW_EVENT_EMITTED` | returns to L0 as new reality introduction event |

### Readiness Regression (Always Active)
| # | Event | Description |
|---|-------|-------------|
| 24 | `REALITY_DRIFT_DETECTED` | triggers readiness review |
| 25 | `READINESS_REGRESSION` | if required: LIVE→SEEDED→ADDING→OFF |
| 26 | `L1_MODULES_LOCK + L2_AUTONOMY_REVOKED` | enforced immediately |

### 2️⃣ Data Source Onboarding

L0 supports multiple onboarding paths:

| Method | Description |
|--------|-------------|
| File Upload | CSV, PDF, JSON |
| OAuth Connector | Salesforce, Google, Slack |
| API Credentials | REST/GraphQL endpoints |
| Manual Mapping | Field-by-field setup |
| Historical Import | Bulk historical data |
| Incremental Sync | Scheduled sync setup |

**All of this is onboarding, not learning yet.**

---

### 3️⃣ Schema Expectation Seeding

L0 seeds **expectations**, not schemas:

```sql
-- L0 says:
"We expect accounts"
"We expect contacts"
"We expect revenue-like fields"

-- L3 will:
Observe, confirm, extend, or contradict
```

**This feeds L3 but does not constrain it.**

---

### 4️⃣ Onboarding UX + Progress Orchestration

L0 owns:

- Setup flows
- Progress indicators
- Error handling
- Retry logic
- User guidance

**This is why L0 is a layer, not a wizard.**

---

### 5️⃣ Onboarding Completion Signals

L0 emits handoff signals to L3:

```
ONBOARDING_STARTED
SOURCE_CONNECTED
INITIAL_DATA_AVAILABLE
HANDOFF_TO_ADAPTIVE_SPINE
```

**After this → L0 is mostly done.**
