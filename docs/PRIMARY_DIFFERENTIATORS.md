## 2. The MuleSoft-Inspired Loader Formula

### Background

MuleSoft pioneered API-led connectivity with:
- **Traits**: Interface contracts (what a system can do)
- **Resource Types**: Canonical data models (what data looks like)
- **Connectors**: Implementation of traits for specific systems

IntegrateWise adapts this for the AI-native era.

### The 8-Stage Loader as MuleSoft Evolution

| Stage | MuleSoft Equivalent | IntegrateWise Enhancement |
|-------|--------------------|---------------------------|
| **ST1: Analyzer** | Input metadata | + AI entity detection |
| **ST2: Classifier** | Type router | + Activity vs Entity classification |
| **ST3: Filter** | Validator | + PII scrubbing + noise removal |
| **ST4: Refiner** | Deduplicator | + Delta detection + merge strategy |
| **ST5: Extractor** | Data mapper | + Schema extraction + field prediction |
| **ST6: Validator** | Schema validation | + Structural + semantic validation |
| **ST7: Sanity** | Business rules | + ML-based anomaly detection |
| **ST8: Sectorizer** | Tenant router | + Scope isolation + permission check |

### The Normalizer as Trait Implementation

```
MuleSoft Pattern:
  Trait "create-contact" implemented by HubSpot connector
  → Maps fields
  → Calls API
  → Returns result

IntegrateWise Pattern:
  Trait "create-contact" implemented by HubSpot connector
  → NA0: Schema detection (what is this data?)
  → NA1: Canonical transformation (normalize fields)
  → NA2: SSOT binding (stable identity)
  → NA3: Lineage tracking (where did this come from?)
  → NA4: Relationship mapping (what is it connected to?)
  → NA5: Dual-write (Truth + Context)
  → Intelligence: Pattern detection + enrichment suggestions
  → Edge correction: Apply learned rules
  → Returns enriched, linked, intelligence-augmented result
```
