# Cloudflare Deployment (Pages)

This repo is intended to deploy on **Cloudflare Pages**.

## Prereqs

- Cloudflare account
- `wrangler` installed (or use `pnpm cf:deploy`)

## Build

Use the Pages-compatible build output generated by `@cloudflare/next-on-pages`:

- `pnpm cf:build`

This will:
1. Run `next build`
2. Generate a Pages/Workers compatible output in `.vercel/output/`

## Deploy

- `pnpm cf:deploy`

This deploys `.vercel/output/static` to Cloudflare Pages.

## Notes

- This project uses Next.js App Router; Pages deployment relies on the generated `.vercel/output` produced by `@cloudflare/next-on-pages`.
- Avoid Node-only APIs in request handlers. Prefer standard Web APIs (`Request`, `Response`, `fetch`, `crypto.subtle`).
- Tenant/plan entitlements are served from `/api/tenant/context` and used client-side for feature gating.

## Local testing

- `pnpm dev`

(Cloudflare Pages emulation via `wrangler pages dev` can be added once the Pages project is created.)
